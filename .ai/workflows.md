# Development Workflows & Observability Runbook

---

## 1. Development Environment Setup

### Prerequisites

- **Python 3.12+** with [uv](https://github.com/astral-sh/uv) package manager
- **Bun 1.0+** for the frontend
- **Docker & Docker Compose** for service dependencies
- **Ollama** running locally (`ollama serve`)
- **PostgreSQL 16+** with pgvector, pg_trgm, pg_bm25 extensions

### Quick Start

```bash
# 1. Infrastructure (Postgres, Redis, Langfuse)
docker compose -f compose.yaml -f compose.dev.yaml up -d

# 2. Pull models
ollama pull gemma4:e4b     # Generation (2-4GB VRAM)
ollama pull nomic-embed-text  # Embeddings

# 3. Backend
cd backend
uv sync
uv run alembic upgrade head
uv run uvicorn src.main:app --reload

# 4. Worker (separate terminal)
cd backend
uv run arq src.workers.arq_worker.WorkerSettings

# 5. Frontend (separate terminal)
cd frontend
bun install
bun run dev
```

Or use `./run.sh` from the repo root for an interactive walkthrough.

## 3. Database Migrations (Alembic)

```bash
# Generate new migration after model changes
uv run alembic revision --autogenerate -m "description"

# Review the generated file in alembic/versions/
# Then apply:
uv run alembic upgrade head

# Rollback one step
uv run alembic downgrade -1
```

**Always review autogenerated migrations** — Alembic can miss index changes and complex column type transitions.

## 4. Background Job Processing

### ARQ Worker (`workers/arq_worker.py`)
- Handles: Document ingestion (parse → chunk → embed → save)
- Queue: Redis
- Job: `process_document(document_id)`
- Concurrency: Controlled by `max_jobs` in `WorkerSettings`

### Memory Worker (`workers/memory_worker.py`)
- Handles: Expired memory cleanup, consolidation tasks
- Schedule: ARQ cron jobs
- Separated from ingestion to avoid blocking document processing

### Flow
```
Client → API → Redis (enqueue_job) → ARQ Worker → LangGraph Ingestion → DB
```

### Monitoring
- Check Redis queue depth: `redis-cli LLEN arq:queue`
- Worker logs show per-job timing
- Langfuse traces show LangGraph node execution times

## 6. Security Operations

1. **Password hashing:** Argon2id via `passlib.hash.argon2` (time_cost=3, memory_cost=65536, parallelism=4)
2. **JWT tokens:** HS256, 30min access + 7d refresh; Redis-backed JTI blacklist for logout
3. **API key encryption:** Fernet symmetric encryption for stored BYOK provider keys
4. **SQL injection prevention:** All queries via SQLAlchemy ORM/parameter binding — never raw string interpolation
5. **Rate limiting:** Token bucket per endpoint — register 5/min, login 10/min, chat 20/min, password ops 5/min

## 7. Observability

### Langfuse
- Access at `http://localhost:3001` (when Docker Compose is running)
- Traces every LLM call with token counts, latency, prompt/response
- LangGraph node executions logged as spans
- Retrieval quality: hit rate, MRR computed in post-processing

### Structured Logging
- JSON-formatted logs via standard Python logging
- Key events: ingestion stage timing, retrieval latency, quality gate decisions, BYOK provider selection

### Key Metrics
- Ingestion throughput (docs/min)
- P50/P95/P99 retrieval latency
- Quality gate pass/fail ratio
- BYOK vs local inference ratio
